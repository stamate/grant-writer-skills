# Supporting Documents

You are generating the supporting documents for a grant proposal, formatted according to agency-specific requirements.

## Arguments

- `--proposal-dir <path>`: Proposal directory (required)
- `--doc <type>`: Specific document to generate (optional — generates all if omitted). One of: `cv`, `facilities`, `dmp`, `ethics`, `consortium`, `letters`, `overlap`

Parse from the user's message.

## Procedure

### 0. Locate Plugin Root

```bash
if [ -f "tools/verify_setup.py" ]; then GRANTWRITER_ROOT="$(pwd)"
else GRANTWRITER_ROOT=$(find ".claude/plugins" "$HOME/.claude/plugins" -maxdepth 8 -name "verify_setup.py" -path "*grant-writer*" 2>/dev/null | head -1 | xargs dirname | xargs dirname); fi
export GRANTWRITER_ROOT; if [ -z "$GRANTWRITER_ROOT" ]; then echo "ERROR: Could not find grant-writer-skills plugin root."; fi; echo "Plugin root: $GRANTWRITER_ROOT"
```

### 1. Load Agency Requirements

```bash
uv run python3 "$GRANTWRITER_ROOT/tools/agency_requirements.py" info <agency> <mechanism>
```

Determine which supporting documents are required or optional for this agency.

### 2. Collect PI Information

**Human checkpoint**: Ask the PI for the information needed to generate the requested documents:
- Full name, affiliation, position, ORCID
- Publication list (top 10 relevant publications with DOIs)
- H-index and total citations
- Previous grants (title, agency, amount, dates, role)
- Available facilities and equipment
- Research data types that will be generated
- Collaborator names and institutions (for letters and consortium)

### 3. Generate Documents

Generate each applicable document and save to `<proposal_dir>/supporting/`.

#### CV (`supporting/cv_pi.md`)

- **EU proposals** (Horizon Europe, ERC, MSCA): Generate Europass-style or ERC template CV. Include: personal info, education, positions, 10 best publications, ongoing grants, supervision track record, key metrics (H-index, total funding).
- **Romanian proposals** (UEFISCDI, PNRR): Generate UEFISCDI-format CV. Include: personal info, education, positions, ISI publications (last 5 years), patents, grants as PI and team member, international experience.

For consortium proposals, generate `cv_partner_N.md` for each partner PI.

#### Facilities & Equipment (`supporting/facilities.md`)

Describe institutional resources available for the project: lab space, computing infrastructure, specialized equipment, libraries, databases, core facilities. Tie each facility to specific project activities.

#### Data Management Plan (`supporting/data_management.md`)

- **EU proposals**: Follow the Horizon Europe DMP template structure — data summary, FAIR data principles, allocation of resources, data security, ethical aspects, other issues.
- **Romanian proposals** (UEFISCDI): Follow UEFISCDI requirements — data types, storage, access policy, long-term preservation. Simpler format than EU DMP.

#### Ethics Self-Assessment (`supporting/ethics_self_assessment.md`)

**Skip this step if** the agency is not EU (ethics self-assessment is EU-specific).

For EU proposals, generate the ethics issues table covering: human participants, personal data (GDPR), human embryos/foetuses, animals, third countries, environment, dual use, AI/ML, other. Mark each as Yes/No with brief justification.

#### Consortium Agreement Outline (`supporting/consortium_agreement.md`)

**Skip this step if** the proposal is single-PI (no consortium partners).

For multi-partner Horizon Europe proposals, draft an outline covering: IP ownership, access rights, decision-making, conflict resolution, financial arrangements, publication policy.

#### Letters of Support (`supporting/letters/`)

Draft template letters of support or collaboration for each collaborator. Include: relationship to PI, relevance of collaboration, specific contribution, commitment statement. Save as `supporting/letters/letter_<name>.md`.

#### Overlap Analysis (`supporting/overlap.md`)

Read `<proposal_dir>/landscape/overlap_analysis.md` (generated by the landscape skill). Reformat as the agency-required overlap/current support document, explicitly stating for each active grant: title, agency, amount, period, and whether there is scientific or budgetary overlap with the proposed work.

### 4. Review Documents

**Human checkpoint**: Present all generated documents for PI review. Key items to verify:
- CV accuracy (publications, dates, metrics)
- Facilities description completeness
- DMP alignment with actual research data plans
- Ethics self-assessment correctness
- Letter of support appropriateness

### 5. Update State

```bash
uv run python3 "$GRANTWRITER_ROOT/tools/state_manager.py" update <proposal_dir> --phase supporting_docs --status complete
```

Report:
- Documents generated (list each with path)
- Any documents skipped and why (e.g., ethics skipped for Romanian proposals)
- Items flagged for PI attention
